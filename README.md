# Recipe Project

Проект выполнен в рамках домашнего задания по теме ООП, тестирования и работы с Git.

## Описание проекта

В проекте реализована простая система для работы с рецептами и списками покупок.

Программа позволяет:

* создавать ингредиенты;
* создавать обычные рецепты;
* добавлять ингредиенты в рецепт;
* масштабировать рецепт на нужное количество порций;
* создавать список покупок на основе одного или нескольких рецептов;
* объединять одинаковые ингредиенты в списке покупок;
* удалять из списка покупок ингредиенты, относящиеся к конкретному рецепту;
* объединять два списка покупок;
* работать с диетическими рецептами.

## Основные классы

### Ingredient

Класс `Ingredient` описывает ингредиент рецепта.

У ингредиента есть три основных атрибута:

* `name` — название ингредиента;
* `quantity` — количество;
* `unit` — единица измерения.

В классе реализована проверка количества: значение `quantity` должно быть положительным. Если передать неположительное значение, будет вызвано исключение `ValueError`.

Также в классе реализованы методы для строкового представления объекта и сравнения ингредиентов по названию и единице измерения.

### Recipe

Класс `Recipe` описывает обычный рецепт.

Рецепт содержит:

* `title` — название рецепта;
* `ingredients` — список ингредиентов.

Класс позволяет добавлять ингредиенты в рецепт. Если ингредиент с таким же названием и такой же единицей измерения уже есть в рецепте, его количество увеличивается, а новый отдельный ингредиент не создаётся.

Также в классе реализован метод `scale`, который создаёт новую версию рецепта с увеличенным или уменьшенным количеством ингредиентов.

### ShoppingList

Класс `ShoppingList` описывает список покупок.

Он хранит ингредиенты с привязкой к рецептам и позволяет:

* добавлять рецепт в список покупок с учётом количества порций;
* удалять из списка ингредиенты конкретного рецепта;
* получать итоговый список ингредиентов для покупки;
* объединять одинаковые ингредиенты;
* объединять два списка покупок с помощью оператора `+`.

### DietaryRecipe

Класс `DietaryRecipe` является наследником класса `Recipe`.

Он дополнительно хранит тип диеты, например:

* вегетарианская;
* без сахара;
* безглютеновая.

При масштабировании диетического рецепта тип диеты сохраняется.

## Запуск программы

Файл `recipes.py` содержит классы проекта и используется как модуль.

Для ручной проверки работы программы можно создать файл `main.py` и импортировать в него нужные классы.

Пример файла `main.py`:

```python
from recipes import Ingredient, Recipe, ShoppingList, DietaryRecipe


lasagna = Recipe("Лазанья")
lasagna.add_ingredient(Ingredient("Листы для лазаньи", 250, "г"))
lasagna.add_ingredient(Ingredient("Фарш мясной говяжий", 450, "г"))
lasagna.add_ingredient(Ingredient("Молоко", 800, "мл"))

shopping_list = ShoppingList()
shopping_list.add_recipe(lasagna, 2)

for ingredient in shopping_list.get_list():
    print(ingredient)

diet_lasagna = DietaryRecipe("Овощная лазанья", "вегетарианская")
diet_lasagna.add_ingredient(Ingredient("Листы для лазаньи", 250, "г"))

print(diet_lasagna)
```

После этого программу можно запустить командой:

```bash
python3 main.py
```

Если используется Windows, команда может быть такой:

```bash
python main.py
```

## Запуск тестов

Для запуска тестов используется библиотека `pytest`.

Запустить тесты можно командой:

```bash
pytest
```

Если тесты проходят успешно, в терминале будет показано количество пройденных тестов.

## Тестирование

В проекте реализованы тесты для основных классов:

* `Ingredient`;
* `Recipe`;
* `ShoppingList`;
* `DietaryRecipe`.

Тесты проверяют:

* создание ингредиентов;
* проверку положительного количества ингредиента;
* сравнение ингредиентов;
* создание рецептов;
* добавление новых ингредиентов;
* объединение одинаковых ингредиентов в рецепте;
* масштабирование рецепта;
* обработку ошибки при некорректном коэффициенте масштабирования;
* создание списка покупок;
* удаление рецепта из списка покупок;
* объединение одинаковых ингредиентов в списке покупок;
* сортировку итогового списка покупок;
* объединение двух списков покупок;
* работу диетических рецептов.
